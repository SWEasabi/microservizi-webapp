Sweasabi Project - Webapp

Overview

This project is a sample Angular application that demonstrates the management of lamps, sensors, and areas. It provides functionality to view the status of these devices, toggle their states, and add new devices. The application utilizes Angular components, services, and API calls to interact with the backend server.

Features

  • View a list of lamps, sensors, and areas.
  • Toggle the status of lamps and sensors.
  • Add new lamps, sensors, and areas.
  • Handle loading and error states during API calls.
  • Implement authentication and protected routes.

Installation

To run this project locally, follow these steps:

  1. Clone the repository: git clone <repository-url>
  2. Navigate to the project directory: cd <project-directory>
  3. Install the dependencies: npm install
  4. Start the development server: npm start

Architecture and Design

Model

The project defines three interfaces to represent the status of lamps, sensors, and areas. These interfaces (LampStatus, SensorStatus, AreaStatus) define the structure of the status objects and provide properties to store relevant information such as status, identifier, and alias.

API Service

The ApiService is responsible for making API requests to interact with the backend server. It utilizes the Angular HttpClient to send HTTP requests and receive responses. The service provides methods to retrieve lamp, sensor, and area data, toggle the status of devices, and add new devices.

Components

  • LampsListComponent: Displays a list of lamps and allows users to toggle their status.
  • AreasListComponent: Displays a list of areas.
  • SensorsListComponent: Displays a sensors of areas.
  • LampButtonComponent: Represents a lamp item in the list with its status and toggle functionality.
  • AreaButtonComponent: Represents an area item in the list.
  • SensorButtonComponent: Represents an area item in the list.

Services

  • AppService: Manages the state of the application by storing the status of lamps, sensors, and areas in a store. It communicates with the ApiService to fetch data, update status, and handle loading and error states. The service also provides methods to add new devices.
  • AuthService: Provides authentication functionality to check if a user is logged in.

Guards

  • AuthGuard: A custom authentication guard that protects routes based on the user's authentication status. If the user is not logged in, it redirects to the login page.

Usage

The application provides an intuitive user interface with a list of lamps and areas. Users can toggle the status of lamps and add new devices. Protected routes require authentication to access.
